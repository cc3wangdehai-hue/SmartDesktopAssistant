## 📋 发现的问题列表

### 问题1: `dotnet build` 不支持 `-o Output` 参数 ⚠️ 高优先级

**影响文件**: 
- 任何使用此参数的脚本

**问题描述**:
用户反馈执行 `dotnet build xxx.sln -o Output` 时构建失败。`dotnet build` 命令本身不支持 `-o` 输出目录参数，此参数仅在 `dotnet publish` 时有效。

**根因分析**:
```
dotnet build xxx.sln -o Output  ❌ 错误
         └─ build命令不支持 -o 参数

dotnet publish xxx.sln -o Output  ✅ 正确（发布模式）
dotnet build xxx.sln -c Release   ✅ 正确（编译到默认目录）
```

**当前状态**: 
检查发现当前 `run.bat` 和 `install.bat` **已使用正确语法**，不存在此问题。

---

### 问题2: 项目引用一致性 ⚠️ 中优先级

**影响文件**: 
- SmartDesktopAssistant.sln
- SmartDesktopAssistant.csproj

**问题描述**:
解决方案文件包含6个项目，但主项目只引用了4个Widget。

**详细信息**:
| 项目 | 在.sln中 | 被主项目引用 |
|------|---------|-------------|
| SmartDesktopAssistant | ✅ | - |
| WeatherWidget | ✅ | ✅ |
| TodoWidget | ✅ | ✅ |
| FilesWidget | ✅ | ✅ |
| SettingsWidget | ✅ | ✅ |
| WidgetsStandalone | ✅ | ❌ (独立版本) |
| ClockWidget | ❌ | ❌ |

**分析**: 
- `ClockWidget` 在 `Widgets/` 目录下存在但**未包含在解决方案中**，且有 `bin/` 和 `obj/` 目录存在，说明可能有部分构建过
- 这可能导致 Widget 复用困难

---

### 问题3: 硬编码输出路径 ⚠️ 低优先级

**影响文件**: 
- install.bat

**问题描述**:
复制命令使用相对路径 `SmartDesktopAssistant\bin\Release\net8.0-windows\`，虽然当前路径正确，但如果项目配置改变可能导致问题。

**改进建议**:
添加路径存在性检查，在复制前验证源文件是否存在。

---

### 问题4: 缺少独立Widget组件 ⚠️ 中优先级

**影响文件**: 
- ClockWidget

**问题描述**:
ClockWidget 存在于 `Widgets/ClockWidget/` 目录，但：
1. 未加入解决方案
2. 未被任何项目引用
3. 只有一个简单的 ClockWindow 控件

**分析**:
这是一个未完成的组件，可能在后续开发中被遗忘。
