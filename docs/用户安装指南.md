## 快速安装

### 方法一：一键安装（推荐）

1. **下载项目源码**
   ```
   下载本项目并解压到任意目录
   ```

2. **运行安装脚本**
   ```
   双击运行: install.bat
   ```

3. **等待完成**
   - 脚本会自动检查环境
   - 克隆源码（如未克隆）
   - 编译所有项目

### 方法二：使用PowerShell

```powershell
# 以管理员身份打开 PowerShell
cd "智能桌面助手_CSharp"
.\install.bat
```

## 手动安装

如果一键安装失败，请按以下步骤手动安装：

### 步骤1：安装 .NET SDK

1. 访问 https://dotnet.microsoft.com/download/dotnet/8.0
2. 下载 **.NET SDK 8.0.x (x64)**
3. 运行安装程序，按提示完成安装

**验证安装**：
```cmd
dotnet --version
# 应显示: 8.0.x
```

### 步骤2：安装 Visual Studio 2022

1. 访问 https://visualstudio.microsoft.com/downloads/
2. 下载 Visual Studio 2022 Community（免费）
3. 运行安装程序
4. 选择工作负载：**.NET 桌面开发**

### 步骤3：克隆源码

打开命令提示符（CMD），执行：

```cmd
cd 智能桌面助手_CSharp

# 克隆 Desktop Fences+
git clone https://github.com/limbo666/DesktopFences.git --depth 1

# 克隆 Desktop Widgets Board
git clone https://github.com/hhlitval/Desktop-Widgets-Board.git --depth 1
```

### 步骤4：打开解决方案

1. 启动 Visual Studio 2022
2. 选择 **打开项目或解决方案**
3. 依次打开以下解决方案：

```
智能桌面助手_CSharp/
├── DesktopFences/Code/Desktop Fences.sln
├── Desktop-Widgets-Board/Desktop Widgets Board.sln
└── ElementHostTest/ElementHostTest.csproj  (直接打开项目)
```

## 运行测试

### 1. 编译所有项目

在 Visual Studio 中：
1. 选择 **生成** → **重新生成解决方案**
2. 等待编译完成
3. 检查输出窗口是否有错误

### 2. 运行 ElementHostTest

1. 在解决方案资源管理器中，右键点击 **ElementHostTest**
2. 选择 **设为启动项目**
3. 按 **F5** 运行

**预期结果**：
- 弹出一个WinForms窗口
- 窗口中显示一个蓝色渐变的天气卡片（WPF控件）
- 可以点击按钮测试控件交互

### 3. 测试 Desktop Fences+

⚠️ **注意**：Desktop Fences+ 会修改桌面图标布局，请在测试前备份重要数据。

1. 在 Visual Studio 中设置 **Desktop Fences** 为启动项目
2. 按 **F5** 运行
3. 桌面上将出现透明的可移动围栏区域

## 常见问题

### Q1: 运行 dotnet 命令提示 "不是内部或外部命令"

**原因**：.NET SDK 未正确安装或PATH未配置

**解决方法**：
1. 重新安装 .NET SDK
2. 重启计算机
3. 或在 VS Developer Command Prompt 中运行

---

### Q2: 编译报错 "找不到 IWshRuntimeLibrary"

**原因**：COM引用缺失

**解决方法**：
1. 在 Visual Studio 中打开项目
2. 右键点击 **引用** → **添加引用**
3. 切换到 **COM** 选项卡
4. 搜索并勾选 **Windows Script Host Object Model**

---

### Q3: ElementHost 显示为白色背景

**原因**：未正确设置透明背景

**解决方法**：
确保在 WinForms 代码中设置：
```csharp
_elementHost.BackColor = Color.Transparent;
```

---

### Q4: WPF控件无法点击

**原因**：窗口焦点问题

**解决方法**：
参考 `NonActivatingWindow.cs` 中的实现，确保：
1. 设置 `WS_EX_NOACTIVATE` 窗口样式
2. 处理 `WM_MOUSEACTIVATE` 消息

---

### Q5: 天气Widget图片不显示

**原因**：图片路径为硬编码的绝对路径

**解决方法**：
将 `MainWindow.xaml` 中的路径改为相对路径：
```xml
<!-- 错误 -->
<Image Source="C:\Users\sasha\Desktop\sunny.png" />

<!-- 正确 -->
<Image Source="Images/sunny.png" />
```
